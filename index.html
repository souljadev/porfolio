Tic-Tac-Toe

User can sign-up, sign-in, play game, keep track of games played, and log out.

This was my very first full stack project. There was a lot of emotions building this project, I've hit a lot of coding walls and I've learned a great deal about how to overcome them. Laying out User Stories and ERD first, then sudo code, and finally move forward to coding.

I first tackled the game logic. I felt that having a strong foundation to build on was important. I wanted the game to work first and then attach it to the database.

I tried to break down the game into smaller problems, and deal with them individually. I created a game board as a base, then add logic to the Xs and Os to make sure the game works. Next I assigned it values and made sure everything worked locally. Then deployed the site and checked to see if it ran like I intended to. After everything worked, I then designed it with James Bond images.

I felt a sense of achievement when the project came together and I got all the logic to work. I started to look back at this project as a reminder, that no matter how big the task may seem, it can be solved just take my time and compartmentalize into smaller tasks.

  • Github: https://github.com/souljadev/tictactoegame
  • Project Site: https://souljadev.github.io/tictactoegame/
  • Technology Used: HTML, CSS3, jQUERY, JavaScript, GIT/GITHUB, SASS

Real Estate CRM

User can create an account, log in, change password, log out. User can input client name and picture. User can see clients data that was inputted. User can log off.

I created a real estate CRM site for users to be able to track their own clients. I used javascript, jQuery, and Ruby on Rails. It was my first time using Rails, Heroku. Learning how relationships work in the backend and building my very own API was a great achievement for me.

This project was smoother to complete because I was able to use planning techniques I've learned through planning from my Tic-tac-toe project.

Message Board

Users can create account, log in, change pw, can add a message, link a picture, see all posts and see their own posts.

This message board is created for Ms. Moy as a retirement present. Ms. Moy has been teaching for 30 years at Boston Public School. I wanted to create a site where her old students can leave her a message.

I decided to use Ember.js on this project because it was responsive and quick. It can render the posts pretty quickly. It was a bit of learning curve, only had a day to learn Ember in class and had to implement it right away. There's still a lot to add to this project. The frustrating thing is Ember.js is an opinionated framework, figuring out how it wants to operate was a bit overwhelming on a tight deadline.

Luckily I was able to meet the deadline. I tackled everything I knew I had control over first, built the backend first. Got all CRUD actions to work so I didn't have too much since the foundation was set. Then all the remaining time I was able to read the Ember docs, and get MVP.

  • Github: https://github.com/souljadev/bmcap
  • Project Site: https://souljadev.github.io/bmcap/
  • Technology Used: HTML, CSS3, Ember.js, Handlebars, Boostrap, SCSS, JavaScript, GIT/GITHUB, SASS, HEROKU, Ruby, Ruby on Rails, MYSQL
